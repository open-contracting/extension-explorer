Requisitos

Agrega una sección de criterios a las secciones de objeto de licitación y respuestas a ofertas, adjudicaciones y contratos. Basado en el criterio principal y el vocabulario de evidencia central (CCCEV) de la UE.

Criterion

La extensión define un nuevo Criterion objeto con estos campos:

Campo Título Descripción Tipo(s)
id Identificador

El identificador para este criterio. Debe de ser único y no puede cambiar dentro del Proceso de Contratación Abierta del cual es parte (definido por un sólo ocid). Ver la guía de identificadores para mayores detalles.

 stringointeger
title Título

Un título para este criterio

 string
description Descripción

Una descripción de este criterio. La información estructurada se debe de dar en los otros campos.

 string
source Fuente

Source of response to the requirements specified in the criterion. For example, responses might be submitted by tenderers or by an assessment committee at the procuringEntity.

 cadena de la lista de código responseSource cerrada
relatesTo Elementos relacionados al esquema

The schema element that the criterion judges, evaluates or assesses. For example, the criterion might be defined against items or against bidders.

 cadena de la lista de código relatesTo cerrada
relatedItem Artículo relacionado

Donde relatesTo = "item" este campo debe de ser llenado con el id del artículo en esta sección de la licitación con el que se relaciona el criterio. Se debe omitir el campo cuando relatesTo <> "item".

 string
requirementGroups Grupos de requisitos

Una lista de grupos de requisitos para este Criterio. Un criterio se satisface cuando uno o más grupos de requisitos se cumplen. Un grupo de requisitos se cumple cuando todos los requerimientos del grupo se satisfacen.

RequirementGroup

La extensión define un nuevo RequirementGroup objeto con estos campos:

Campo Título Descripción Tipo(s)
id Identificador

El identificador para este grupo de requisitos. Debe ser único y no puede cambiar dentro del Proceso de Contratación Abierta del que forma parte (definido por un solo ocid). Consulte la guía de identificadores para obtener más detalles.

 string
description Descripción

Una descripción de este grupo de requisitos. Se debe de proveer información estructurada en la lista de requisitos.

 string
requirements Requisitos

Una lista de requisitos que deben ser cumplidos para que el grupo de requisitos se cumpla.

Requirement

La extensión define un nuevo Requirement objeto con estos campos:

Campo Título Descripción Tipo(s)
id Identificador del requisito

El identificador de este requisito. Debe ser único y no puede cambiar dentro del Proceso de Contratación Abierta de que forma parte (definido por un solo ocid). Consulte la guía de identificadores para obtener más detalles.

 string
title Título de requisito

El título de este requisito atómico.

 string
description Descripción del requisito

Una descripción de texto libre para este requisito atómico

 string
dataType Tipo de dato de Requisito

El tipo de dato en el que se debe proporcionar la respuesta del requisito.

 cadena de la lista de código dataType cerrada
pattern Patrón de validación de respuesta

Una expresión regular para validar la respuesta requerida

 string
expectedValue Valor esperado

Usado para que se establezca el requisito cuando la respuesta debe ser un valor particular.

 stringointegeronumberoboolean
minValue Valor mínimo

Se usa para indicar el límite inferior del requisito cuando la respuesta debe estar dentro de un cierto rango.

 integeronumber
maxValue Valor máximo

Se usa para indicar el límite superior del requisito cuando la respuesta debe estar dentro de un cierto rango.

 integeronumber
period Periodo

Usado para especificar un período particular al que el requisito aplica, por ejemplo, el volumen de negocio del licitante en un año específico.

RequirementResponse

La extensión define un nuevo RequirementResponse objeto con estos campos:

Campo Título Descripción Tipo(s)
id Identificador

El identificador para este requisito de respuesta. Debe de ser único y no puede cambiar dentro del Proceso de Contratación Abierta del cual es parte (definido por un sólo ocid). Ver la guía de identificadores para mayores detalles.

 string
title Título

Un título para la respuesta del requisito.

 string
description Descripción

Una descripción de este grupo de requisitos. Se debe de proveer información estructurada en la lista de requisitos.

 string
value Valor

El valor de esta respuesta de requisito. El valor debe ser el tipo que se define en el campo requirement.data.

 stringointegeronumberoboolean
period Periodo

El período en que se aplica la respuesta de requisitos.

 Period objetos
requirement Requisito relacionado

El id y título que aplica a la respuesta de requisitos.

 RequirementReference objetos
relatedTenderer Licitador relacionado

Donde esta respuesta de requisito se relaciona con un licitador y es provista por el comprador o la entidad compradora, este campo debe usarse para referenciar la entrada en la sección de partes para el licitador al cual se relaciona la respuesta.
